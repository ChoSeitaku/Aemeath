# Aemeath 开发计划

> 鸣潮爱弥斯主题个人AI助手 - 完整开发路线图

## 项目概述

**项目名称**：Aemeath（爱弥斯/小爱）
**项目类型**：个人AI助手（陪伴型）
**技术栈**：TypeScript + Ink + Bun / DeepSeek V4 Flash / Whisper / Edge TTS
**预计工期**：15-20周
**目标**：打造一个基于鸣潮爱弥斯角色的个人AI助手，具备对话、工具、语音、多模态、陪伴功能。

#### 工具基类设计

```python
from abc import ABC, abstractmethod
from typing import Any, Dict
from pydantic import BaseModel

class ToolInput(BaseModel):
    """工具输入基类"""
    pass

class ToolOutput(BaseModel):
    """工具输出基类"""
    success: bool
    data: Any = None
    error: str = None

class BaseTool(ABC):
    """工具基类"""
    
    name: str
    description: str
    parameters: Dict[str, Any]
    
    @abstractmethod
    async def execute(self, **kwargs) -> ToolOutput:
        """执行工具"""
        pass
    
    def get_schema(self) -> Dict:
        """获取工具 Schema"""
        return {
            "type": "function",
            "function": {
                "name": self.name,
                "description": self.description,
                "parameters": self.parameters
            }
        }
```

### Phase 7: 多设备同步（第7-8周）

**目标**：实现跨设备数据同步，支持多平台无缝切换。

> 详细设计请查看 [docs/MULTI_DEVICE_SYNC.md](docs/MULTI_DEVICE_SYNC.md)

#### 设计原则
1. **本地优先（Local-First）**：所有数据优先存储在本地，同步是可选的增强功能
2. **离线可用**：无网络时所有功能正常工作，联网后自动同步
3. **冲突友好**：冲突不是错误，而是需要解决的状态
4. **隐私安全**：同步数据端到端加密，用户完全控制
5. **无缝体验**：切换设备时对话上下文自然延续

```bash
# 1. 克隆或创建仓库
git init aemeath
cd aemeath

# 2. 创建虚拟环境
python -m venv venv

venv\Scripts\activate  # Windows
```

## 开发规范

### Git提交规范

```
feat: 添加新功能
fix: 修复bug
docs: 更新文档
style: 代码格式调整
refactor: 重构代码
test: 添加测试
chore: 构建/工具变更
```

### 代码规范

- 使用 `black` 格式化代码
- 使用 `isort` 排序导入
- 使用 `mypy` 进行类型检查
- 遵循 PEP 8 规范

### 测试要求

- 每个功能完成后立即编写测试
- 测试覆盖率不低于 80%
- 使用 `pytest` 运行测试
